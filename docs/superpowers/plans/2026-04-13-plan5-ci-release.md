# CI, WASM Embedding & Release Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Wire up the WASM fallback backend and ship GitHub Actions workflows for CI, native addon builds, and npm publishing.

**Architecture:** Five tasks in dependency order: (1) generate the embedded WASM files locally and commit them, (2) implement the `wasm.ts` adapter that initialises the WASM blob and wraps it as a `Backend`, (3) refactor `loader.ts` to pass `algorithms` to `loadWasm` and use the real WASM adapter, (4) add the CI check workflow, (5) add the build-matrix and release workflows plus finalise the npm package config.

**Tech Stack:** Rust, wasm-bindgen 0.2.118 (`--target web`), Bun 1.3, Biome, GitHub Actions, cargo-llvm-cov 0.8+, napi-rs

- [ ] **Step 1: Create the build script**

`scripts/build-wasm.sh`:
```bash
#!/usr/bin/env bash
set -euo pipefail

# Builds hashjunkie-wasm and generates the embedded TypeScript blob.
# Run whenever crates/hashjunkie-wasm/src/lib.rs changes.
#
# Prerequisites:
#   rustup target add wasm32-unknown-unknown
#   cargo install wasm-bindgen-cli --version 0.2.118
#   wasm-opt (optional): apt install binaryen / brew install binaryen

SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
REPO_ROOT="$(cd "$SCRIPT_DIR/.." && pwd)"
PKG_DIR="$REPO_ROOT/npm/hashjunkie"
CARGO_HOME="${CARGO_HOME:-$HOME/.cargo}"
export PATH="$CARGO_HOME/bin:$PATH"
WASM_RAW="$REPO_ROOT/target/wasm32-unknown-unknown/wasm-release/hashjunkie_wasm.wasm"

echo "==> Building hashjunkie-wasm..."
cargo build \
  --manifest-path "$REPO_ROOT/Cargo.toml" \
  -p hashjunkie-wasm \
  --target wasm32-unknown-unknown \
  --profile wasm-release

# Optional: shrink with wasm-opt
if command -v wasm-opt &>/dev/null; then
  echo "==> Running wasm-opt -O3..."
  wasm-opt -O3 "$WASM_RAW" -o "$WASM_RAW"
else
  echo "==> wasm-opt not found — skipping (install binaryen for smaller output)"
fi

# Generate JS glue with wasm-bindgen --target web
TMPDIR_OUT=$(mktemp -d)
trap 'rm -rf "$TMPDIR_OUT"' EXIT
echo "==> Running wasm-bindgen --target web..."
wasm-bindgen --target web --out-dir "$TMPDIR_OUT" "$WASM_RAW"

# Copy glue files into the npm package
cp "$TMPDIR_OUT/hashjunkie_wasm.js"   "$PKG_DIR/hashjunkie_wasm.js"
cp "$TMPDIR_OUT/hashjunkie_wasm.d.ts" "$PKG_DIR/hashjunkie_wasm.d.ts"

# Generate wasm_blob.ts — base64 of the .wasm binary
BLOB=$(base64 -w0 "$TMPDIR_OUT/hashjunkie_wasm_bg.wasm")
cat > "$PKG_DIR/wasm_blob.ts" << TSEOF
// Generated by scripts/build-wasm.sh — do not edit manually.
// Regenerate when crates/hashjunkie-wasm/src/lib.rs changes:
//   ./scripts/build-wasm.sh
export const WASM_BLOB = "$BLOB";
TSEOF

echo "==> Done. Generated:"
echo "    $PKG_DIR/hashjunkie_wasm.js"
echo "    $PKG_DIR/hashjunkie_wasm.d.ts"
echo "    $PKG_DIR/wasm_blob.ts ($(wc -c < "$PKG_DIR/wasm_blob.ts") bytes)"
```

```ts
const ABC_SHA256 = "ba7816bf8f01cfea414140de5dae2223b00361a396177a9cb410ff61f20015ad";
```

`npm/hashjunkie/wasm.ts`:
```ts
import { initSync, WasmHasher } from "./hashjunkie_wasm.js";
import type { Algorithm, Backend, Digests } from "./types";
import { WASM_BLOB } from "./wasm_blob";

let initialized = false;

function ensureInit(): void {
  if (initialized) return;
  const bytes = Uint8Array.from(atob(WASM_BLOB), (c) => c.charCodeAt(0));
  // initSync with { module: bytes } wraps bytes in WebAssembly.Module synchronously.
  // No filesystem access — bytes come from the inline base64 blob.
  initSync({ module: bytes });
  initialized = true;
}

export function makeWasmBackend(algorithms: Algorithm[]): Backend {
  ensureInit();
  const hasher = new WasmHasher(algorithms);
  return {
    update(data: Uint8Array): void {
      hasher.update(data);
    },
    finalize(): Digests {
      // Trust assertion: WasmHasher.finalize() returns exactly the requested
      // algorithm keys mapping to lowercase hex strings — same guarantee as
      // the Rust MultiHasher it wraps.
      return hasher.finalize() as Digests;
    },
  };
}
```

### Task 3: Loader Refactor for WASM Algorithms

**Files:**

`loadWasm` must receive `algorithms` so `makeWasmBackend` can create the right hasher.

- [ ] **Step 1: Update `loader.ts`**

Apply these changes to `npm/hashjunkie/loader.ts`:

**Add import at the top** (after the existing `import type { Algorithm, Backend, Digests }` line):
```ts
import { makeWasmBackend } from "./wasm";
```

**Change the `Loaders` type** (around line 70):
```ts
// OLD:

  loadNative: () => NativeAddon | null;
  loadWasm: () => Backend | null;
};

// NEW:
type Loaders = {
  loadNative: () => NativeAddon | null;
  loadWasm: (algorithms: Algorithm[]) => Backend | null;
};
```

**Replace `_defaultLoadWasm`** (currently `return null`):
```ts
/** Returns a WASM backend for the given algorithms. Throws if WASM initialisation fails. */
export function _defaultLoadWasm(algorithms: Algorithm[]): Backend | null {
  return makeWasmBackend(algorithms);
}
```

**Update `loadBackend`** — pass `algorithms` to `loadWasm`:
```ts
// OLD:
const wasm = _loaders.loadWasm();

// NEW:
const wasm = _loaders.loadWasm(algorithms);
```

The full updated `loadBackend` function:
```ts
export function loadBackend(algorithms: Algorithm[]): Backend {
  const addon = _loaders.loadNative();
  if (addon !== null) {
    const inst = new addon.NativeHasher(algorithms);
    return {
      update(data: Uint8Array): void {
        inst.update(Buffer.from(data));
      },
      finalize(): Digests {
        // Trust assertion: the Rust layer always returns exactly the 13 Algorithm keys
        return inst.finalize() as Digests;
      },
    };
  }

  const wasm = _loaders.loadWasm(algorithms);
  if (wasm !== null) return wasm;

  throw new Error(
    "hashjunkie: no backend available — native addon failed to load and WASM initialisation failed",
  );
}
```


- [ ] **Step 2: Write `.github/workflows/ci.yml`**

```yaml
name: CI

on:
  push:
    branches: [main]
  pull_request:

env:
  CARGO_TERM_COLOR: always
  RUST_BACKTRACE: 1

jobs:
  rust:
    name: Rust checks
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Install Rust toolchain
        uses: dtolnay/rust-toolchain@stable
        with:
          components: rustfmt, clippy, llvm-tools-preview

      - name: Cache Cargo
        uses: Swatinem/rust-cache@v2

      - name: Check formatting
        run: cargo fmt --all -- --check

      - name: Clippy (warnings as errors)
        # hashjunkie-napi uses napi-rs macros that generate code incompatible with
        # cfg(test) — exclude it from the workspace clippy run.
        run: >
          cargo clippy
          --workspace
          --exclude hashjunkie-napi
          --all-targets
          -- -D warnings

      - name: Tests
        run: >
          cargo test
          --workspace
          --exclude hashjunkie-napi

      - name: Install cargo-llvm-cov
        uses: taiki-e/install-action@cargo-llvm-cov

      - name: Coverage (100% gate)
        run: >
          cargo llvm-cov
          --workspace
          --exclude hashjunkie-napi
          --branch
          --fail-under-lines 100

  typescript:
    name: TypeScript checks
    runs-on: ubuntu-latest
    defaults:
      run:
        working-directory: npm/hashjunkie
    steps:
      - uses: actions/checkout@v4

      - name: Set up Bun
        uses: oven-sh/setup-bun@v2
        with:
          bun-version: latest

      - name: Install dependencies
        run: bun install

      - name: Biome lint + format check
        run: ./node_modules/.bin/biome ci .

      - name: Tests with coverage (100% gate)
        # Parse bun's coverage table and fail if any file is below 100%.
        run: |
          bun test --coverage 2>&1 | tee /tmp/coverage.txt
          if grep -E "[[:space:]]+[0-9]{1,2}\.[0-9]{2}[[:space:]]" /tmp/coverage.txt | grep -v "100\.00"; then
            echo "Coverage below 100% in one or more files."
            exit 1
          fi
```

```yaml
          - target: x86_64-unknown-linux-gnu

            node-file: hashjunkie.linux-x64-gnu.node

            node-file: hashjunkie.darwin-x64.node

            runner: macos-latest
            node-file: hashjunkie.darwin-arm64.node
          - target: x86_64-pc-windows-msvc
            runner: windows-latest
            node-file: hashjunkie.win32-x64-msvc.node

    steps:
      - uses: actions/checkout@v4

      - name: Install Rust toolchain
        uses: dtolnay/rust-toolchain@stable
        with:
          targets: ${{ matrix.target }}

      - name: Cache Cargo
        uses: Swatinem/rust-cache@v2

        run: cargo install cross --git https://github.com/cross-rs/cross

      - name: Build (native)

        run: >
          cp target/${{ matrix.target }}/release/libhashjunkie_napi.dylib

        run: >
          Copy-Item
          target/${{ matrix.target }}/release/hashjunkie_napi.dll
```

- [ ] **Step 5: Write `.github/workflows/release.yml`**

Triggered when a GitHub Release is published. Downloads build artifacts, places `.node` files into each platform package, then publishes all packages to npm.

```yaml
name: Release

on:
  release:
    types: [published]

jobs:
  publish:
    name: Publish to npm
    runs-on: ubuntu-latest
    permissions:
      contents: read
      id-token: write  # for npm provenance
    steps:
      - uses: actions/checkout@v4

      - name: Set up Bun
        uses: oven-sh/setup-bun@v2
        with:
          bun-version: latest

      - name: Download all native addon artifacts
        uses: actions/download-artifact@v4
        with:
          pattern: "hashjunkie.*.node"
          merge-multiple: true
          path: /tmp/native-artifacts

      - name: Place .node files into platform packages
        run: |
          cp /tmp/native-artifacts/hashjunkie.linux-x64-gnu.node   npm/hashjunkie-linux-x64-gnu/
          cp /tmp/native-artifacts/hashjunkie.linux-arm64-gnu.node  npm/hashjunkie-linux-arm64-gnu/
          cp /tmp/native-artifacts/hashjunkie.darwin-x64.node       npm/hashjunkie-darwin-x64/
          cp /tmp/native-artifacts/hashjunkie.darwin-arm64.node     npm/hashjunkie-darwin-arm64/
          cp /tmp/native-artifacts/hashjunkie.win32-x64-msvc.node   npm/hashjunkie-win32-x64-msvc/

      - name: Configure npm token
        run: echo "//registry.npmjs.org/:_authToken=${{ secrets.NPM_TOKEN }}" > ~/.npmrc

      - name: Publish platform packages
        run: |
          for pkg in \
            npm/hashjunkie-linux-x64-gnu \
            npm/hashjunkie-linux-arm64-gnu \
            npm/hashjunkie-darwin-x64 \
            npm/hashjunkie-darwin-arm64 \
            npm/hashjunkie-win32-x64-msvc; do
            echo "Publishing $pkg..."
            cd "$pkg"
            npm publish --provenance --access public
            cd -
          done

      - name: Publish main package
        working-directory: npm/hashjunkie
        run: npm publish --provenance --access public
```

Note: this workflow requires a GitHub secret `NPM_TOKEN` (an npm automation token) and the build-native workflow must have already run for this release tag to produce the artifacts.
